Before / After

Before — Plan→Ask→Auto で3つのMode通知が個別表示
After — 連続切替しても “Mode → auto” の1行のみ

Bug 1: Mode通知の連続表示

再現手順: Modeボタンをクリック → Plan選択 → Ask選択 → Auto選択

再現結果: 3つの "Mode → plan", "Mode → ask", "Mode → auto" が縦に並ぶ

修正: showSystemNotice() で直前の同種noticeを検索し、テキストを置き換え。4箇所の呼び出しを統一

Bug 2: Mode通知が画面下部に固定

原因: ストリーミング中に appendChild で assistant メッセージの後に追加 → scrollToBottom で常に最下部に表示

修正: tab.sending が true の場合、insertBefore(el, streamingAssistant) で assistant メッセージの手前に挿入

確認: ストリーミング中のMode切替でnoticeがメッセージフロー内の自然な位置に表示されることを確認
