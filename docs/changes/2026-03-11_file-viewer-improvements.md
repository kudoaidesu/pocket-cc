# ファイルビューアー改善: バツボタン・形式別表示・作成リンク

- **Issue**: ファイル選択時のUX改善（バツボタン・MD表示・ファイルリンク）
- **日付**: 2026-03-11
- **ブランチ**: develop
- **プロジェクト**: pocket-cc

## TL;DR

| 項目 | 内容 |
|------|------|
| **課題** | ファイルビューアーのバツボタンがファイル名テキストに被って押せない、MDがプレーンテキスト表示で読みにくい |
| **変更内容** | バツボタンのテキスト被り解消（min-width:0 + 44px化）、ファイル形式別レンダリング、Write/Edit後のファイルリンク追加 |
| **影響範囲** | Web UIのファイルビューアー全般 |
| **リスク** | Low — UI変更のみ、既存API・ロジックへの影響なし |
| **切り戻し** | git revert で即時復元可能 |

## 変更内容

### 変更ファイル

| ファイル | 変更種別 | 概要 |
|---------|---------|------|
| `src/web/public/index.html` | 修正 | バツボタンCSS、ファイル形式別レンダリング、ファイルリンクUI |
| `src/web/server.ts` | 修正 | 画像ファイル配信用 `/api/files/raw` エンドポイント追加 |

### 設計判断

| 判断 | 代替案 | 理由 | トレードオフ |
|------|--------|------|------------|
| 既存 `marked.js` でMDレンダリング | highlight.js等の追加ライブラリ | 既にプロジェクトで使用中、追加依存なし | シンタックスハイライトは未対応（行番号のみ） |
| `<pre>` → `<div>` にコンテナ変更 | 条件分岐でpre/div切替 | MD時のブロック要素レンダリングに必須 | CSSクラスでモード切替が必要 |
| 画像は `/api/files/raw` 新エンドポイント | Base64インライン | 大きい画像でもストリーム配信可能、5MB制限付き | エンドポイント1つ追加 |
